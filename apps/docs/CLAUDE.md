# `apps/docs` — Docusaurus documentation site

## Purpose

External-facing documentation site for the boilerplate. Runs on port `3100` in dev (leaving `3000` for `apps/web`). Builds to a static site (`build/`) that can be served from anywhere — the default deploy target is a Vercel project under a `docs.` subdomain.

Distinct from `knowledge/` — that's the Obsidian-style internal vault (ADRs, decision logs, architecture notes). This site is for public consumers of the template.

## Entry points

- `docusaurus.config.ts` — site metadata, nav bar, footer, theme, preset-classic config.
- `sidebars.ts` — autogenerated sidebar (adds any MD file under `docs/`).
- `docs/*.md` — public guides (getting-started, architecture, deploy).
- `blog/` — release notes / announcements.
- `src/pages/index.tsx` — custom landing page.
- `src/css/custom.css` — Infima theme overrides.

## Architectural rules

1. **Independent React version.** Docusaurus 3.x pins to React 18; `apps/web` uses React 19. pnpm workspaces isolate the installs — never hoist React to the root and never share a single React copy between the two apps.
2. **Static output.** No runtime server component — every page is prerendered at build time. Don't pull in anything that requires a Node runtime at request time.
3. **Content lives here.** Docs MDX files are authored in this app, not copied from `knowledge/`. The two are separate by design: `knowledge/` is for LLMs and maintainers; `docs/` is for end users.
4. **No mutable env at runtime.** `docusaurus.config.ts` reads env vars at build time only (e.g. `DOCS_SITE_URL`). Don't reach for `process.env` inside page components.

## Forbidden patterns

- Importing from `packages/*` — the docs app must build standalone (enables independent deploys).
- Server-rendered components — breaks the static-export assumption.
- Pulling content from `knowledge/` via symlinks or build steps — those docs serve a different audience. Repetition is acceptable when the wording differs.

## Common tasks

- **Dev:** `pnpm --filter @startup-boilerplate/docs dev` → http://localhost:3100
- **Build:** `pnpm --filter @startup-boilerplate/docs build` → `apps/docs/build/`
- **Serve built site:** `pnpm --filter @startup-boilerplate/docs start`
- **Clear Docusaurus cache:** `pnpm --filter @startup-boilerplate/docs clear`
- **Add a doc:** drop an MD file under `docs/`; the autogenerated sidebar picks it up. Set `sidebar_position` in front matter to control order.
- **Add a blog post:** create `blog/YYYY-MM-DD-slug.md` with front matter matching the existing posts.

## Testing requirements

- Build must succeed (`pnpm --filter @startup-boilerplate/docs build`) with `onBrokenLinks: "throw"` — internal link rot is a build-time error.
- Typecheck passes (`tsc --noEmit`).
