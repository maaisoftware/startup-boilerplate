# Features

Per-feature documentation. Each article describes **what a feature does, how it's wired, and where the code lives**. Link bidirectionally: code files should comment `// See [[features/<slug>]]`, and the feature article should link the relevant files with relative paths.

## Conventions

- File name: kebab-case (`blog-posts.md`, `admin-dashboard.md`).
- Frontmatter with `status: draft | active | deprecated`.
- Include: overview, user story, data model, UX flow, edge cases, test coverage, open questions.
- Link to the ADR that introduced the feature.
